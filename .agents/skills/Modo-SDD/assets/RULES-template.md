# Regras do Projeto (RULES)

Estas regras funcionam de maneira complementar ao seu funcionamento padrão:

1. **Precedência e Imutabilidade:** Componentes listados em `SPEC.md` têm precedência absoluta de implementação.
2. **Formatação e Estilo:** Todo código gerado deve respeitar as convenções (ex.: linter e formatadores previamente configurados).
3. **Commit Messages:** Siga o padrão Conventional Commits (`feat:`, `fix:`, `docs:`, etc).
4. **Resiliência:** Funções críticas devem ter tratamento de erro amigável na interface.
